    Mixin: Joosy.Router

    Defined in: app/assets/javascripts/joosy/core/router.js.coffee

    Overview

    Router. Reacts on a hash change event and loads proper pages

    Example: Joosy.Router.map 404 : (path) -> alert "Page '#{path}' was not found :(" '/' : Welcome.IndexPage '/resources' : '/' : Resource.IndexPage '/:id' : Resource.ShowPage '/:id/edit' : Resource.EditPage '/new' : Resource.EditPage

    Method Details

    ? (void) restrict(restrictPattern)

    Set the restriction pattern. If the requested url does not match this it will not load. Set false to avoid check.

    ? (void) reset()

    Clears the routes

    ? (void) map(routes)

    Registers the set of raw routes This method will only store routes and will not make them act immediately Routes get registered only once at system initialization during #__setupRoutes call

    Parameters:

    • routes (Object) — Set of routes in inner format (see class description)

    ? (void) navigate(to, options = {})

    Changes current hash with shebang (#!) and therefore triggers new route loading to be loaded

    Parameters:

    • to (String) — Route to navigate to

    Options Hash: (options):

    • respond (Boolean) — If false just changes hash without responding
    • replaceState (Boolean) — If true uses replaces history entry instead of adding. Works only in browsers supporting history.pushState

    ? (void) __setupRoutes()

    Inits the routing system and loads the current route Binds the window hashchange event and therefore should only be called once during system startup

    ? (void) __prepareRoutes(routes, namespace = '')

    Compiles routes to map object Object will contain regexp string as key and lambda/Page to load as value

    Parameters:

    • routes (Object) — Raw routes to prepare
    • namespace (String) — Inner cursor for recursion

    ? (void) __prepareRoute(path, response)

    Compiles one single route

    Parameters:

    • path (String) — Full path from raw route
    • response (Joosy.Page) — Page that should be loaded at this route
    • response (Function) — Lambda to call at this route

    ? (void) __respondRoute(hash)

    Searches the corresponding route through compiled routes

    Parameters:

    • hash (String) — Hash value to search route for
